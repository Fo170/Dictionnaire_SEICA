1. 1             GENERAL-PURPOSE RESOURCES ON THE ACL DSP MODULE

On the ACL module (ACL DSP module) there are some general-purpose hardware resources present on the two connectors “USER” (JP6) and “FIXTURE” (JP5) shown in Fig. 410. From: Fig. 411 to: Fig. 416 display the electric schemes of the signals present on the two connectors “USER” (JP6) and “FIXTURE” (JP5).

The resources available are:

 

OC0-OC7                    group of 8 outputs “OPEN COLLECTOR”          (3 mA, 10 KHz Max).

RTOIS0- RTOIS3       group of 4 optoisolated outputs                     (10 mA, 2 MHz Max).

JTOIS0-JTOIS1         group of 2 optoisolated outputs                     (5 mA, 2 MHz Max).

FXO0-FXO3    group of 4 optoisolated outputs                     (3 mA, 10 KHz Max).

RTI0-RTI3                 group of 4 optoisolated inputs                                   (5 mA, 2 MHz Max).

JTI0-JTI3                  group of 4 optoisolated inputs                                   (5 mA, 2 MHz Max).

FXI0-FXI3                 group of 4 optoisolated inputs                                   (5 mA, 10 KHz Max).

 

INSTRUMENT WR_HW for operations of writing of the hardware output resources.

STRUMENTO  RD_HW           for operations of reading of the hardware input resources.

 

 

 

1.1.1 [WR_HW]

 

Syntax

VIVA LANGUAGE

 

~SET WR_HD   [TYPE=OC|RT|JT|FX| [DATA=value]

 

Detail of parameters

 

TYPE=option|value      [OC|RT|JT|FX , default: OC]

Defines the type of output hardware resource which will be set

OC | 0                      Identifies the 8 outputs “open collector”, (OC0OC7).

RT | 1                       identifies the 4 “optoisolated”, (RTOIS0-RTOIS3).

JT | 2                       Identifies the two “optoisolated” outputs, (JTOIS0JTOIS1).

FX | 3                       Identifies the 4 “optoisolated” outputs, (FXO0FXO3).

 

DATA=value                 [0255, default: 0]

Expresses the configuration of the group selected with parameter “TYPE”. Specifically:

If TYPE=OC       the bit0-bit7, of the byte containing the value of the parameter DATA, correspond to signals “OC0-OC7”. A bit set to 1 defines the corresponding circuit “open collector” opened; a bit set to 0 defines the corresponding circuit “open collector” closed.

If TYPE=RT       The bit0-bit3, of the byte containing the value of the paramter DATA, define the configuration of the 4 output signals: RTOIS0-RTOIS3.

If TYPE=JT        The bit0-bit3, of the byte containing the value of the parameter DATA, define the configuration of the two output signals: JTOIS0 e JTOIS1.

The bit0-bit1 define the value of the signal JTOIS0, with the following meaning:

00       The signal JTOIS0 is maintained to logic level “0”

01       The signal JTOIS0 is maintained to logic level “1”

10       The signal JTOIS0 is connected to the sensor of the digital channel 0.

11       The signal JTOIS0 is connected to the sensor of the digital channel 1.

The bit2-bit3 define the value of the signal JTOIS1, with the following meaning:

00       The signal JTOIS1 is maintained to logic level “0”

01       The signal JTOIS1 is maintained to logic level “1”

10       the signal JTOIS1 is connecterd to the sensor of the digital channel 2.

11       The signal JTOIS1 is connected to the sensor of the digital channel 3.

Se TYPE=FX     the bit0-bit3, of the byte containing the value of the parameter DATA, define the configuration of the four output signals: FXO0-FXO3.

1.1.2  [RD_HD]

METHOD MEASURE

The method MEAS is used to read the hardware input resources.

 

Syntax

VIVA LANGUAGE

 

~MEAS RD_HD [INTO=variable] [TYPE=RT|JT|FX|]

 

 

 

Detail of parameters

 

INTO=variable              Indicates the variable where are transferred the data measured by the general purpose input instruments, present on the ACL DSP MODULE. According to the value of the parameter “TYPE, the measured value will have the following meanings:

If TYPE=RT       the bit0-bit3 of the measured value, represent the configuration of the four input signals: RTI0-RTI3.

If TYPE=JT        the bit0-bit3 of the measured value represent the configuration of the four input signals: JTI0- JTI3.

If TYPE=FX       the bit0-bit3 of the measured value, represent the configuration of the four input signals: FXI0-FXI3.

 

TYPE=option|value      [RT|JT|FX , default: RT]

Defines the source of the data:

RT | 1              The data come from signals RTI0RTI3.

JT | 2              The data come from signals JTI0-JTI3.

FX | 3              The data come from signals FXI0-FXI3.
